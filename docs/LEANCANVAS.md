# Lifeboard (라이프보드) - Lean Canvas

> **작성일**: 2026-02-14 | **버전**: v1.0 | **기반 문서**: PRD v1.0

---

## 1. Problem (문제)

- **정보 분산**: 뉴스, 일정, 건강, 재무 등 일상 정보가 여러 앱/사이트에 흩어져 있어 한눈에 파악 불가
- **뉴스 중복 소비**: 같은 사건을 다룬 여러 기사를 반복적으로 읽으며 시간 낭비
- **의견과 사실 혼재**: 뉴스 기사에 기자/전문가의 주관적 의견이 섞여 있어 사실(팩트) 파악이 어려움

### 기존 대안
- 포털 뉴스 (네이버/다음): 중복 기사 나열, 의견/사실 구분 없음
- RSS 리더 (Feedly 등): 기사 원문만 제공, 요약/그룹핑 없음
- 뉴스레터 (뉴닉/데일리바이트): 편집자 주관 개입, 실시간성 부족

---

## 2. Customer Segments (고객 세그먼트)

- **얼리어답터**: 정보 효율성을 중시하는 IT/비즈니스 직장인 (25~40대)
- 매일 아침 뉴스를 확인하지만 중복 기사에 피로감을 느끼는 사용자
- 사실 기반 정보만 빠르게 파악하고 싶은 사용자
- 여러 정보 소스를 하나의 대시보드에서 관리하고 싶은 사용자

---

## 3. Unique Value Proposition (고유 가치 제안)

> **"의견은 빼고, 팩트만 한눈에"**

AI가 여러 언론사의 뉴스를 자동으로 그룹핑하고 사실(팩트)만 추출하여 제공하는 개인 라이프 대시보드.
중복 기사를 하나로 묶고, 주관적 의견을 걸러낸 핵심 사실만 불릿 포인트로 정리하여 정보 소비 시간을 획기적으로 단축.

---

## 4. Solution (솔루션)

- **RSS 자동 수집**: 한국 주요 언론사 12개 RSS 피드에서 1시간마다 뉴스 자동 수집
- **중복 기사 그룹핑**: 제목 유사도 기반으로 같은 사건을 다룬 기사를 하나의 그룹으로 자동 병합
- **AI 팩트 요약**: 로컬 LLM(Ollama)으로 의견/추측을 제거하고 검증 가능한 사실만 3~5개 추출
- **통합 대시보드**: 뉴스를 카테고리별로 필터링하고 팩트 요약 + 원문 링크를 카드 형태로 제공

---

## 5. Channels (채널)

- **웹앱**: Vercel 배포 (PC/모바일 반응형)
- **직접 접근**: 북마크/즐겨찾기를 통한 일상적 접속
- **향후 확장**: Slack/웹 푸시 알림으로 중요 뉴스 실시간 전달

---

## 6. Revenue Streams (수익 흐름)

- **MVP 단계**: 무료 (개인 프로젝트)
- **향후 고려**:
  - 프리미엄 구독 (고급 분석, 개인화 기능)
  - 추가 데이터 소스 연동 (해외 뉴스, 금융 API)

---

## 7. Cost Structure (비용 구조)

| 항목 | 비용 | 비고 |
|------|------|------|
| Supabase | 무료 티어 | DB + 인증 + Realtime |
| Vercel | 무료 티어 | 호스팅 + Cron 작업 |
| Ollama (로컬 LLM) | 전기세만 | 별도 PC에서 qwen2.5:14b 구동 |
| RSS 피드 | 무료 | API 키 불필요 |
| 도메인 | 연 ~2만원 | 선택사항 |

**핵심**: 외부 AI API(OpenAI 등)를 사용하지 않고 로컬 LLM으로 AI 비용 제로화

---

## 8. Key Metrics (핵심 지표)

- **일간 활성 사용자(DAU)**: 대시보드 방문 횟수
- **뉴스 수집 성공률**: RSS 수집 파이프라인 안정성 (목표: 95%+)
- **AI 요약 완료율**: 생성된 그룹 대비 요약 완료 비율 (목표: 90%+)
- **평균 체류 시간**: 사용자가 대시보드에서 정보를 소비하는 시간
- **원문 클릭률**: 팩트 요약 후 원문 기사로 이동하는 비율

---

## 9. Unfair Advantage (경쟁 우위)

- **로컬 LLM 기반 제로 코스트 AI**: 외부 API 의존 없이 자체 AI 인프라로 운영 비용 최소화
- **팩트 중심 차별화**: 기존 뉴스 서비스와 달리 의견/추측을 제거한 사실만 제공하는 독자적 접근
- **확장 가능한 라이프 데이터 허브**: 뉴스를 시작으로 일정, 건강, 재무 등 다양한 생활 데이터를 통합할 수 있는 플랫폼 구조
